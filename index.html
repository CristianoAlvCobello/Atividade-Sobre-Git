Git e GitHub? Como Funciona? Instalação

O que é Git?

Git é um sistema de controle de versão distribuído, gratuito e de código aberto, projetado para gerenciar desde projetos pequenos até muito grandes com rapidez e eficiência.

Git é fácil de aprender e possui uma pegada mínima com desempenho extremamente rápido. Ele supera ferramentas SCM como Subversion, CVS, Perforce e ClearCase, com recursos como ramificação local barata, áreas de preparação convenientes e múltiplos fluxos de trabalho.

Fonte: Site do Git.

O que é Controle de Versão?

O controle de versão é a prática de rastrear e gerenciar alterações no código de software. Sistemas de controle de versão armazenam todas as mudanças em um banco de dados, permitindo que desenvolvedores voltem no tempo para corrigir erros. Eles são essenciais para equipes de software de alto desempenho e DevOps, proporcionando eficiência e agilidade.

Esses sistemas permitem um histórico completo de alterações, incluindo quem fez cada alteração e por quê. Eles suportam a ramificação e a mesclagem de diferentes fluxos de trabalho, permitindo que equipes trabalhem simultaneamente em várias funcionalidades sem conflitos. Ferramentas como o Git, um VCS distribuído e gratuito, facilitam a identificação e correção de bugs, a colaboração em equipe e a manutenção do código ao longo do tempo, o que é crucial para o desenvolvimento contínuo e o gerenciamento eficaz do software.

Além disso, a rastreabilidade detalhada das mudanças permite conectar alterações a sistemas de gestão de projetos e rastreamento de bugs, ajudando na análise de causa raiz e garantindo que as modificações estejam alinhadas com o design de longo prazo do sistema. Isso é especialmente importante para trabalhar com código legado e estimar com precisão o trabalho futuro.

Fonte: Atlassian.

O que são Branches?

Branches (ramificações) no Git são versões independentes de desenvolvimento dentro de um repositório. Elas permitem que você trabalhe em diferentes funcionalidades ou correções de bugs ao mesmo tempo, sem afetar a linha principal de desenvolvimento. Cada branch pode ser criada, modificada e mesclada de volta à branch principal (geralmente chamada de "main" ou "master") quando estiver pronta. Isso facilita o gerenciamento de várias tarefas e colaborações em projetos complexos.

Fonte: Site do Git.

O que são Git pull?

O comando git pull é usado para sincronizar o conteúdo de um repositório local com as alterações feitas em um repositório remoto. Ele combina dois passos: primeiro, busca as atualizações remotas usando git fetch, e depois mescla essas alterações com o repositório local usando git merge. Em resumo, o git pull garante que o conteúdo do repositório local esteja atualizado com o repositório remoto, criando um novo commit de merge se necessário.

Fonte: Atlassian.

O que é GitHUB?

O GitHub está intimamente ligado ao Git, pois é uma plataforma que facilita a gestão de código e a colaboração entre desenvolvedores, utilizando o Git como sistema de controle de versão. Ela simplifica o uso do Git, escondendo alguns detalhes de configuração. Normalmente, é onde você hospedará e gerenciará seu repositório.

Embora seja possível editar arquivos diretamente na interface web do GitHub, isso não é recomendado, pois não oferece um ambiente completo de desenvolvimento e testes. Para interagir com o GitHub, você pode usar comandos Git como `commit`, `pull` e `push`.

Se isso parecer complicado, você pode optar pelo GitHub Desktop, uma aplicação mais intuitiva para gerenciar seu código.

Fonte: Alura.

O que é GitLAB?

GitLab é uma plataforma de DevOps que oferece uma interface web para gerenciamento de repositórios Git, integrando várias funcionalidades de desenvolvimento de software em um único aplicativo. Isso inclui controle de versão, CI/CD (Integração Contínua/Entrega Contínua), planejamento de projetos, revisão de código, rastreamento de issues, e muito mais. É uma alternativa ao GitHub, com um forte enfoque em automação e colaboração para aumentar a eficiência das equipes de desenvolvimento.

Fonte: GitLab
